# 📌 K-Nearest Neighbors (KNN) — Complete Guide

---

## 📖 Overview

**K-Nearest Neighbors (KNN)** is a simple, intuitive, and powerful supervised machine learning algorithm used for:

* **Classification**
* **Regression**

It is a **lazy learning** and **instance-based** algorithm, meaning it does not build a model during training — it stores the data and makes predictions during inference.

---

## 🚀 Key Characteristics

| Feature         | Description                 |
| --------------- | --------------------------- |
| Type            | Supervised Learning         |
| Learning Style  | Lazy Learning               |
| Model Type      | Non-parametric              |
| Usage           | Classification & Regression |
| Training Time   | Very Low                    |
| Prediction Time | High                        |

---

## 🧠 How KNN Works

### Step-by-Step Process

1. Choose the number of neighbors (**K**)
2. For a new data point:

   * Calculate distance from all training points
3. Select the **K nearest neighbors**
4. Make prediction:

   * **Classification** → Majority vote
   * **Regression** → Average of values

---

## 📊 Example

### Dataset

| Age | Salary | Buy Product |
| --- | ------ | ----------- |
| 25  | 30k    | No          |
| 30  | 40k    | Yes         |
| 35  | 60k    | Yes         |

### New Data Point

```
Age = 28
Salary = 35k
```

### Prediction Process

* Calculate distance from all points
* Pick nearest neighbors (K=3)
* Output → Majority class = **Yes**

---

## 📏 Distance Metrics

### 1. Euclidean Distance (Most Common)

```
d = √[(x1 - x2)^2 + (y1 - y2)^2]
```

### 2. Manhattan Distance

```
d = |x1 - x2| + |y1 - y2|
```

### 3. Minkowski Distance

```
d = (|x1 - x2|^p + |y1 - y2|^p)^(1/p)
```

---

## ⚙️ Choosing the Value of K

| K Value | Effect                      |
| ------- | --------------------------- |
| Small K | High variance (overfitting) |
| Large K | High bias (underfitting)    |

### Rule of Thumb

```
K ≈ √(number of data points)
```

---

## ❓ Why KNN is Non-Parametric

KNN is **non-parametric** because:

* It does **not assume any data distribution**
* It does **not learn parameters** like weights
* It simply **stores the dataset** and uses it directly

### One-Line Definition

> KNN is non-parametric because it does not build a model — it uses the entire dataset during prediction.

---

## 📌 Advantages

* Simple and easy to understand
* No training phase
* Works well with small datasets
* No assumptions about data distribution

---

## ⚠️ Disadvantages

* Slow for large datasets
* Sensitive to noise and outliers
* Requires feature scaling
* High memory usage

---

## 📐 Feature Scaling (IMPORTANT)

KNN is distance-based, so scaling is critical.

### Example

| Feature | Range          |
| ------- | -------------- |
| Age     | 20–60          |
| Salary  | 20,000–100,000 |

👉 Salary dominates distance → Wrong predictions

### Solution

* Standardization
* Normalization

## 💻 Python Implementation

```python
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import KNeighborsClassifier
from sklearn.metrics import accuracy_score

# Sample Data
X = np.array([[25,30000],[30,40000],[35,60000],[40,70000]])
y = np.array([0,1,1,1])

# Train-Test Split
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# Feature Scaling
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)
X_test = scaler.transform(X_test)

# Model
knn = KNeighborsClassifier(n_neighbors=3)
knn.fit(X_train, y_train)

# Prediction
y_pred = knn.predict(X_test)

# Accuracy
print("Accuracy:", accuracy_score(y_test, y_pred))
```

## 🧠 Interview Answers (Detailed)

### 1. Why is KNN called lazy learning?

KNN is called **lazy learning** because it **does not learn a model during training**. Instead, it simply stores the training data and **performs computation only at prediction time**.

> In one line: KNN delays learning until prediction, hence it is called lazy.

---

### 2. Why is KNN sensitive to scaling?

KNN relies on **distance calculations**. If features have different scales, larger-scale features dominate the distance.

Example:

* Age: 20–60
* Salary: 20,000–100,000

👉 Salary will dominate distance → incorrect neighbors → poor predictions

> Solution: Use **Standardization or Normalization**

---

### 3. What happens when K is too large or too small?

| K Value               | Effect                                          |
| --------------------- | ----------------------------------------------- |
| Too Small (e.g., K=1) | High variance, overfitting, sensitive to noise  |
| Too Large             | High bias, underfitting, ignores local patterns |

> Balance is required to get optimal performance.

---

### 4. Why is KNN non-parametric?

KNN is **non-parametric** because:

* It does **not assume any distribution** of data
* It does **not learn parameters (weights, coefficients)**
* It uses the **entire dataset directly** for prediction

> In one line: No model parameters → non-parametric.

---

### 5. How to choose optimal K?

#### Methods:

1. **Rule of Thumb**

```
K ≈ √n
```

2. **Cross-Validation (Best Method)**

* Try multiple K values
* Choose K with highest accuracy

3. **Odd Value of K**

* Helps avoid ties in classification

4. **Elbow Method (Accuracy vs K graph)**

* Choose K where accuracy stabilizes

---

## 📎 Final Tip

> Always combine **feature scaling + cross-validation** to get the best KNN performance.
