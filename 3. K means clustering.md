## 1️⃣ What is K-Means Clustering?

**K-Means** is an **unsupervised machine learning algorithm** used to group similar data points into **K clusters**.

* **K** = number of clusters (user-defined)
* Goal = minimize distance within clusters and maximize distance between clusters

👉 In simple words:

> It groups similar data points together based on distance.

---

## 2️⃣ Intuition

Imagine you have student marks and want to group them into:

* Low performers
* Average performers
* High performers

Instead of manually labeling, **K-Means automatically finds these groups**.

---

## 3️⃣ Step-by-Step Working of K-Means

---

### Step 1: Choose K (Number of Clusters)

* Decide how many clusters you want
* Example: K = 3

---

### Step 2: Initialize Centroids

* Randomly select K data points as centroids
* These are initial guesses

---

### Step 3: Assign Data Points

* Calculate distance from each point to all centroids
* Assign to nearest centroid

#### Euclidean Distance Formula

```math
d = \sqrt{(x_1 - x_2)^2 + (y_1 - y_2)^2}
```

---

### Step 4: Update Centroids

* Compute mean of all points in each cluster
* New centroid = average of cluster points

---

### Step 5: Repeat

Repeat Step 3 and 4 until:

* Centroids stop changing OR
* Maximum iterations reached

---

### Step 6: Final Output

* Stable clusters
* Final centroids

---

## 4️⃣ Objective Function (WCSS)

K-Means minimizes:

```math
WCSS = \sum_{i=1}^{K} \sum_{x \in C_i} (x - \mu_i)^2
```

Where:

* (C_i) = cluster
* (\mu_i) = centroid

👉 Goal: **Minimize total intra-cluster distance**

---

## 5️⃣ Choosing Optimal K

---

### 🔹 Method 1: Elbow Method

#### Steps:

1. Run K-Means for K = 1 to 10
2. Compute WCSS for each K
3. Plot graph (K vs WCSS)

#### Interpretation:

* WCSS decreases as K increases
* Find point where reduction slows

👉 That point = **Elbow Point**

---

### 🔹 Method 2: Silhouette Score

* Range: **-1 to 1**
* Higher score = better clustering

## 7️⃣ Disadvantages

| Disadvantage                | Explanation                       |
| --------------------------- | --------------------------------- |
| Need K beforehand           | Must choose number of clusters    |
| Sensitive to initialization | Different results each run        |
| Local minima                | May not find global optimum       |
| Poor for complex shapes     | Works only for spherical clusters |
| Sensitive to outliers       | Outliers distort centroids        |
| Scaling required            | Distance-based algorithm          |

## 8️⃣ Important Concepts

---

### 🔹 Feature Scaling

K-Means uses distance → scale matters

#### Example:

* Salary: 0–100000
* Age: 20–60

👉 Salary dominates distance → incorrect clusters

#### Solution:

* Standardization (Z-score)
* Min-Max scaling

---

### 🔹 K-Means++

Improves centroid initialization:

* Avoids poor random selection
* Faster convergence
* More stable clustering

---

## 9️⃣ Time Complexity

```math
O(n \cdot k \cdot i \cdot d)
```

Where:

* n = number of data points
* k = number of clusters
* i = number of iterations
* d = number of features

---

## 🔟 When NOT to Use K-Means

Avoid using K-Means when:

* Clusters are non-spherical (e.g., circular shapes)
* Clusters have different densities
* Data contains many outliers

## 1️⃣2️⃣ Python Implementation

```python
from sklearn.cluster import KMeans

# Sample data
X = [[1,2], [1,4], [1,0],
     [10,2], [10,4], [10,0]]

# Apply K-Means
kmeans = KMeans(n_clusters=2, random_state=0)
kmeans.fit(X)

print("Centroids:", kmeans.cluster_centers_)
print("Labels:", kmeans.labels_)
```

---

## 1️⃣3️⃣ Elbow Method Code

```python
from sklearn.cluster import KMeans
import matplotlib.pyplot as plt

wcss = []

for i in range(1, 11):
    kmeans = KMeans(n_clusters=i, random_state=0)
    kmeans.fit(X)
    wcss.append(kmeans.inertia_)

plt.plot(range(1, 11), wcss)
plt.xlabel("Number of clusters")
plt.ylabel("WCSS")
plt.title("Elbow Method")
plt.show()
```

## 1️⃣4️⃣ Final Summary

* K-Means is an unsupervised clustering algorithm
* Groups data into K clusters based on similarity
* Works using iterative centroid updates
* Objective: minimize WCSS
* Optimal K found using Elbow Method
* Works best for well-separated, spherical clusters

## ✅ Key Takeaways

* Always scale your data before applying K-Means
* Use Elbow Method or Silhouette Score to choose K
* Prefer K-Means++ for better initialization
* Avoid using K-Means for complex-shaped data
